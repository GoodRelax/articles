右上に「Copy」ボタンが表示され、クリックすると「Copied!」に変わるか確認してください。

## 1. JavaScript の例
言語指定あり（`javascript`）のパターンです。

```javascript
const greet = (name) => {
  console.log(`Hello, ${name}!`);
};

greet("World");

2. Python の例
別の言語（python）でも正しく動作するか確認します。
def fibonacci(n):
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    else:
        return fibonacci(n-1) + fibonacci(n-2)

print(fibonacci(10))

3. 設定ファイル (YAML/CSS) の例
背景色や文字色が異なる場合でもボタンが見やすいか確認してください。
/* デザインのテスト */
body {
  background-color: #f4f4f4;
  font-family: 'Helvetica Neue', Arial, sans-serif;
}

4. 長い行のテスト (横スクロール)
コードが横に長くなった場合、ボタンの位置がずれないか確認します。
# 長いコマンドのテスト
curl -X POST [https://api.example.com/v1/users](https://api.example.com/v1/users) -H "Content-Type: application/json" -d '{"username": "testuser", "email": "test@example.com", "password": "verysecurepassword123", "bio": "This is a very long text to test horizontal scrolling functionality in the code block component."}'

5. 言語指定なし
言語を指定せずにバッククォート3つだけで囲った場合です。
これはプレーンテキストです。
言語指定がない場合でも <pre> タグが生成されるため、
ボタンが表示されるはずです。

### 確認手順

1.  このファイルを `_posts/2026-01-28-code-test.md` として保存。
2.  GitHubへプッシュ。
3.  数分後、ブログのトップページまたは記事一覧から「コードブロックとコピーボタンのテスト」という記事を開く。
4.  各コードブロックの右上にボタンが出ているかチェックする。

うまくいかない場合は、開発者ツール（F12キー）の「Console」タブにエラーが出ていないか確認してみてください。
